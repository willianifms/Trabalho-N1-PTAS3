TRABALHO DE PROGRAMAÇÃO E TECNOLOGIAS PARA APLICAÇÕES SERVIDOR 3

Tecnologias:

  • Express.JS

    O Express.js é um framework web para Node.js, projetado para simplificar o desenvolvimento de aplicativos web e APIs. Ele serve como uma camada intermediária entre o servidor e o cliente, permitindo que os desenvolvedores criem rapidamente rotas, manipulem solicitações HTTP, gerenciem middleware e muito mais. Em resumo, o Express.js é usado para: Roteamento, Middleware, Manipulação de Requisições e Respostas, Gerenciamento de Recursos Estáticos e Integração com outras bibliotecas. Nós o utilizamos no nosso trabalho nos arquivos: env. (para criar o servidor web e lidar com solicitações HTTP), userController.js (para criar rotas e manipular solicitações HTTP), routes.js (para criar uma API web e criar um conjunto de rotas (ou endpoints) para gerenciar operações relacionadas a usuários em uma aplicação.) e server.js (para fazer a configuração básica de um servidor web).

  • PostgreSQL

    O PostgreSQL, frequentemente abreviado como "Postgres," é um sistema de gerenciamento de banco de dados relacional de código aberto. Ele é um dos sistemas de banco de dados mais avançados e populares disponíveis e é amplamente usado em aplicativos devido à sua confiabilidade, desempenho e recursos avançados. Nós o utilizamos no nosso trabalho nos arquivos: env. (declarando o nome do banco de dados, o nome de usuário e a senha para acessar o banco, o endereço IP do host onde o banco de dados está hospedado, a porta que o banco de dados está usando para aceitar conexões e o ambiente de execução da aplicação), userController.js ( para interagir com o banco de dados, permitindo a criação, leitura, atualização e exclusão de registros),

  • Sequelize CLI

    O Sequelize CLI é uma ferramenta de linha de comando (Command Line Interface) que é usada em conjunto com o Sequelize, um ORM (Object-Relational Mapping) para bancos de dados SQL em JavaScript, especialmente popular no contexto de desenvolvimento de aplicativos Node.js. O Sequelize CLI facilita a interação com o Sequelize e ajuda os desenvolvedores a gerenciar suas bases de dados e modelos de forma mais eficiente. É principalmente usado para: Geração de Modelos, Geração de Migrações, Configuração Simplificada, Seed de Dados e facilita a Manutenção do Banco de Dados. Nós o utilizamos no nosso trabalho nos arquivos: config.js (é um arquivo de configuração para uma aplicação Node.js que utiliza o Sequelize como ORM (Object-Relational Mapping) para interagir com o banco de dados), connection.js (foi usado para autenticar a conexão com o banco de dados usando as informações de configuração fornecidas), userController.js (foi usado para interagir com o banco de dados, permitindo a criação, leitura, atualização e exclusão de registros), nos arquivos de migração (para interagir com o banco de dados e fazer modificações na estrutura da tabela chamada "user") e nos arquivos model ( para lidar com operações de banco de dados SQL).

    • Endpoints:

    • '/user/:id' (update)
